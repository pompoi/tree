# Task: Remove Decorative Branch Hexes

## Description

Remove the three separate decorative branch hexagons (ATK/MOV/DEF) from the center of the graph. The three base skills (Melee Attack, Move, Defend) should take their place as the touching triangle anchors. This eliminates the redundant visual layer where both a decorative hex AND a base skill hex exist per branch.

Currently there are 6 hexagons at center: 3 decorative branch hexes at `CORE_POSITIONS` + 3 base skill hexes positioned by `computeHexLayout()` at BASE_RADIUS=160. After this task: only 3 hexagons (the base skills), touching at corners like the branch hexes do now.

   - Verify LAYER_SPACING=90 still works (depth-1 at ~131px should have no overlap with BRANCH_HEX_R=36 hexes)

### Geometry

For flat-top hexagons (radius R=36) touching at corners in a triangle centered at origin:
- Attack base skill: `(0, -(2*R*√3)/3)` ≈ `(0, -41.6)`
- Movement base skill: `(R, (R*√3)/3)` ≈ `(36, 20.8)`
- Defend base skill: `(-R, (R*√3)/3)` ≈ `(-36, 20.8)`

The radial layout positions base skills on hex ring edges at `BASE_RADIUS`. To get them at ~41.6px from center, set `BASE_RADIUS ≈ 42` (the exact position depends on the edge midpoint formula in the layout algorithm).

**Important**: The radial layout places skills at hex edge midpoints, not at the radius directly. Test visually and adjust BASE_RADIUS until base skills visually touch at corners.
