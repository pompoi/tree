# Task: Base Skill Advantage Borders

## Description

Apply gold/purple border styling to the three base skill hexes (Melee Attack, Move, Defend) based on the hovered skill's branch advantage relationships. These base skills are the triangle anchors and should react visually to reinforce the YOMI triangle.

## Acceptance Criteria

- [ ] When hovering an Attack skill: Move base hex gets a gold border (#fbbf24), Defend base hex gets a purple border (#a855f7)
- [ ] When hovering a Movement skill: Defend base hex gets gold border, Melee Attack base hex gets purple border
- [ ] When hovering a Defend skill: Melee Attack base hex gets gold border, Move base hex gets purple border
- [ ] When hovering a skill of the SAME branch as a base skill: that base skill shows its normal hover glow (not gold/purple)
- [ ] The gold/purple borders are thicker than normal (strokeWidth ~3) and use the glow filters from Task 002
- [ ] Borders clear on mouse leave
- [ ] `bun run build` passes with 0 errors

## Technical Details

### Files Affected

1. **`src/components/skill-tree/UnifiedSkillGraph.tsx`** — only file changed

### Implementation

The base skills are already rendered in the unified skill node render loop (same as all other skills). The `advantageIds` and `disadvantageIds` sets from Task 002 already include base skills. So this task is primarily about ensuring the base skill styling code correctly applies gold/purple.

After Task 001, base skills are rendered in the same loop as all other skills. The advantage/disadvantage styling from Task 002 should already apply to them. This task verifies that and handles edge cases:

1. **Same-branch hover**: When hovering "Sharpen" (attack branch), "Melee Attack" is in the prereq chain (white glow), NOT gold/purple — prereq chain takes priority. Verify this works.

2. **Base skill hover**: When hovering "Melee Attack" itself, it should trigger gold on Move and purple on Defend — verify the `advantageIds`/`disadvantageIds` computation includes base skill IDs.

3. **Visual weight**: Base skills as triangle anchors should have slightly more prominent gold/purple borders than regular skills. Consider using `strokeWidth: 3.5` on base skills vs `strokeWidth: 3` on regular skills for advantage borders.

If special treatment is needed, add a check in the render loop:
```typescript
const isBase = skill.isBase;
if (isAdvantage && !showGlow) {
  stroke = ADVANTAGE_COLOR;
  strokeW = isBase ? 3.5 : 3;
  // ...
}
```
