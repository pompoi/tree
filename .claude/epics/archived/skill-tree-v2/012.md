# Task: Advantage Highlighting Engine

## Description

Add YOMI advantage/disadvantage highlighting to all skill nodes. When hovering any skill, all visible skills of the branch it beats glow **gold** (#fbbf24) and all visible skills of the branch that beats it glow **purple** (#a855f7). This is the core feature of the V2 epic.

## Acceptance Criteria

- [ ] `BEATS` and `LOSES_TO` constant maps added (Attack→Movement, Movement→Defend, Defend→Attack)
- [ ] When hovering an Attack skill: all visible Movement skills get gold highlight, all visible Defend skills get purple highlight
- [ ] When hovering a Movement skill: all visible Defend skills get gold, all visible Attack skills get purple
- [ ] When hovering a Defend skill: all visible Attack skills get gold, all visible Movement skills get purple
- [ ] The hovered skill itself retains its current highlight style (not gold/purple)
- [ ] Prerequisite chain highlighting (white glow) takes priority over gold/purple
- [ ] Confirmed selection outlines render on top of gold/purple
- [ ] Gold/purple clears immediately on mouse leave
- [ ] Works in both Skill Tree and Selection Wheel modes
- [ ] Cross-branch skills (with `secondaryBranch`) highlight based on primary branch
- [ ] `bun run build` passes with 0 errors

### Implementation

Add constants at top of file:

```typescript
const BEATS: Record<Branch, Branch> = {
  attack: "movement",
  movement: "defend",
  defend: "attack",
};
const LOSES_TO: Record<Branch, Branch> = {
  attack: "defend",
  movement: "attack",
  defend: "movement",
};

const ADVANTAGE_COLOR = "#fbbf24"; // gold
const DISADVANTAGE_COLOR = "#a855f7"; // purple
```

Compute advantage sets as derived state (useMemo):

```typescript
const { advantageIds, disadvantageIds } = useMemo(() => {
  if (!hoveredSkill) return { advantageIds: new Set<string>(), disadvantageIds: new Set<string>() };
  const beatenBranch = BEATS[hoveredSkill.branch];
  const beatingBranch = LOSES_TO[hoveredSkill.branch];
  const adv = new Set<string>();
  const dis = new Set<string>();
  for (const node of nodes.values()) {
    if (node.skill.id === hoveredSkill.id) continue;
    if (node.skill.branch === beatenBranch) adv.add(node.skill.id);
    if (node.skill.branch === beatingBranch) dis.add(node.skill.id);
  }
  return { advantageIds: adv, disadvantageIds: dis };
}, [hoveredSkill, nodes]);
```

In the unified skill node render loop, add gold/purple styling:

```typescript
// After base styling, before prereq chain override:
const isAdvantage = advantageIds.has(skill.id);
const isDisadvantage = disadvantageIds.has(skill.id);

if (isDisadvantage && !showGlow) {
  stroke = DISADVANTAGE_COLOR;
  strokeW = 3;
  fill = DISADVANTAGE_COLOR;
  fillOpacity = 0.15;
}
if (isAdvantage && !showGlow) {
  stroke = ADVANTAGE_COLOR;
  strokeW = 3;
  fill = ADVANTAGE_COLOR;
  fillOpacity = 0.15;
}
// showGlow (prereq chain) still overrides both
```

### Visual Priority (highest to lowest)

1. Confirmed outline (fat border, existing)
2. Prereq chain highlight (white glow, existing `showGlow`)
3. Disadvantage (purple stroke + faint fill)
4. Advantage (gold stroke + faint fill)
5. Base fill state (filled/outline based on unlock + mode)
