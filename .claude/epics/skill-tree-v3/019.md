# Task: Progressive reveal in Build mode

## Description

Change Build mode to only render skills that are either unlocked or currently unlockable (all prerequisites met). Hidden skills are simply not rendered — no placeholder hexes. This makes the tree grow organically as the player makes choices.

## Acceptance Criteria

- [ ] Fresh build (only base skills unlocked) shows exactly 15 visible hexes: 3 T0 (filled) + 12 T1 (outlined/available)
- [ ] Unlocked skills render with filled hex (current behavior)
- [ ] Available skills (prereqs met, not yet unlocked) render with hollow hex, dashed stroke, branch color
- [ ] Skills whose prerequisites are NOT all met are not rendered at all
- [ ] Clicking an available hex unlocks it (existing `unlockSkill` logic)
- [ ] After unlocking a skill, newly available skills animate into view (use existing `hex-appear` animation)
- [ ] Prerequisite edges only drawn between visible skills
- [ ] Boost passives (sharpen, swift-feet, iron-skin) appear as available immediately (their T0 prereq is always unlocked)
- [ ] Play mode is NOT affected by this change — Play mode shows only unlocked skills (existing behavior)
- [ ] YOMI wedge triangles still render in the center regardless of reveal state
- [ ] Pan/zoom auto-fits to visible hexes (viewBox adjusts as tree grows)

## Technical Details

### Files affected
- `src/components/skill-tree/UnifiedSkillGraph.tsx` — Add visibility filtering logic

### Implementation approach

1. Compute visibility sets in the component:
```typescript
const unlockedSet = new Set(builds[activeSlot]?.unlockedSkillIds ?? []);
const availableSet = new Set(
  SKILLS.filter(s => !unlockedSet.has(s.id) && s.prerequisites.every(p => unlockedSet.has(p)))
    .map(s => s.id)
);
const visibleSet = new Set([...unlockedSet, ...availableSet]);
```

2. In the hex rendering loop, skip skills not in `visibleSet` (for Build mode)
3. For available (not unlocked) skills, render with:
   - Transparent fill
   - Dashed stroke (`strokeDasharray="4 3"`)
   - Branch color stroke at 50% opacity
   - Slightly smaller scale or pulsing animation to invite interaction
4. Keep the click handler: clicking an available skill calls `unlockSkill`
5. Prerequisite edge rendering: only draw edges where both endpoints are in `visibleSet`

### ViewBox auto-fit
After computing visible hexes, calculate bounding box of their pixel positions and adjust the SVG viewBox to fit with padding. This ensures the tree auto-zooms as it grows.
