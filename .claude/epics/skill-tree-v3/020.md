# Task: Single skill selection in Play mode

## Description

Change Play mode confirm to select exactly one skill instead of the entire prerequisite chain. Currently, confirming a skill adds all ancestors to `confirmedSkillIds` via `getPrereqChainIds()`. The new behavior: confirm sets only the selected skill.

This is a small, isolated change.

## Acceptance Criteria

- [ ] Tapping CONFIRM in ActionBar sets `confirmedSkillIds` to `[selectedSkillId]` (array of 1)
- [ ] Tapping CONFIRM in DesktopSidePanel sets `confirmedSkillIds` to `[selectedSkillId]` (array of 1)
- [ ] `getPrereqChainIds` function call removed from confirm flow (function can stay for other uses)
- [ ] Only the selected skill shows the confirmed visual state (green outline), not ancestors
- [ ] RESET still clears everything via `resetPlayState()`
- [ ] Target hex picker still works as before
- [ ] Confirmed state text shows only the single skill name: "CONFIRMED: Lunge -> D5"

## Technical Details

### Files affected
- `src/components/skill-tree/ActionBar.tsx` — Remove chain logic from `handleConfirm`
- `src/components/skill-tree/UnifiedSkillGraph.tsx` — Remove chain logic from DesktopSidePanel confirm

### Implementation approach

**ActionBar.tsx** — In `handleConfirm`:
```typescript
// BEFORE:
const chain = getPrereqChainIds(skill.id);
chain.push(skill.id);
setConfirmedSkills(chain);

// AFTER:
setConfirmedSkills([skill.id]);
```

**UnifiedSkillGraph.tsx** — In DesktopSidePanel confirm handler, same change.

The `getPrereqChainIds` function at the bottom of ActionBar.tsx can be left in place or removed — it's not used elsewhere.

## Dependencies

None — this change is independent of skill data or layout changes.
