# Task: Skill types and data definitions

## Description

Update the `Skill` TypeScript interface and rewrite `skills.ts` to contain all 57 skills with Fibonacci-weighted prerequisites. This is the foundation task — every other task depends on this data being correct.

**Key changes:**
1. Add `isBoostPassive` field to `Skill` interface in `src/types/skill.ts`
2. Update 9 existing T2 skills to have 2 T1 prerequisites each
3. Add 9 new cross-branch T2 skills (AD, AM, DM hybrids)
4. Remove 6 old T3 skills (executioner, whirlwind, mirror-guard, fortress, phantom-step, overrun)
5. Add 24 new T3 skills across 6 zones
6. Mark sharpen, swift-feet, iron-skin with `isBoostPassive: true`

## Acceptance Criteria

- [ ] `Skill` interface has `isBoostPassive?: boolean` field
- [ ] `SKILLS` array contains exactly 57 entries
- [ ] 3 T0 base skills unchanged (melee-attack, move, defend)
- [ ] 9 active T1 skills unchanged in mechanics, unchanged prerequisites
- [ ] 3 boost T1 skills have `isBoostPassive: true`
- [ ] 9 within-branch T2 skills have exactly 2 T1 prerequisites each per PRD FR-1.1 table
- [ ] 9 cross-branch T2 skills added per PRD FR-1.2 tables (counterstrike, bait-and-punish, step-through, blitz, hit-and-run, ambush, intercept, phaseout, shield-advance)
- [ ] 24 T3 skills added per PRD FR-1.3 tables with correct T2+T1 prerequisites
- [ ] 6 old T3 skills removed (executioner, whirlwind, mirror-guard, fortress, phantom-step, overrun)
- [ ] All skills have complete metadata: id, name, tier, branch, actionType, hexRange, description, statBonus, statBonusAmount, cooldown, prerequisites, isBase
- [ ] Cross-branch T2 skills have `secondaryBranch` set correctly
- [ ] `SKILL_MAP` and `BASE_SKILL_IDS` exports still work
- [ ] `bun run build` passes type checks (may have missing hex-pattern/position errors — expected, fixed in later tasks)

## Technical Details

### Files affected
- `src/types/skill.ts` — Add `isBoostPassive` field
- `src/data/skills.ts` — Major rewrite of SKILLS array

### Implementation approach
1. Update the `Skill` interface first
2. Keep T0 and T1 sections largely intact (only add `isBoostPassive` to boosts)
3. Rewrite T2 section: update prerequisites for 9 existing, add 9 new cross-branch
4. Rewrite T3 section: remove 6 old, add 24 new organized by zone
5. Organize file with clear section comments by tier and branch/zone

### Skill ID naming convention
- Cross-branch T2: kebab-case descriptive (counterstrike, bait-and-punish, step-through, etc.)
- T3: kebab-case descriptive (killshot, blinding-combo, smoke-and-daggers, etc.)

### Cross-branch T2 skill metadata reference

**AD hybrids (branch: nearest primary, secondaryBranch: other):**
- counterstrike: branch="defend", secondaryBranch="attack", actionType="dodge", hexRange=1, statBonus="reduction", statBonusAmount=2
- bait-and-punish: branch="defend", secondaryBranch="attack", actionType="dodge", hexRange=0, statBonus="reduction", statBonusAmount=2
- step-through: branch="attack", secondaryBranch="defend", actionType="melee", hexRange=1, statBonus="damage", statBonusAmount=2

**AM hybrids:**
- blitz: branch="attack", secondaryBranch="movement", actionType="melee", hexRange=2, statBonus="damage", statBonusAmount=2
- hit-and-run: branch="attack", secondaryBranch="movement", actionType="melee", hexRange=1, statBonus="damage", statBonusAmount=2
- ambush: branch="movement", secondaryBranch="attack", actionType="move", hexRange=2, statBonus="movement", statBonusAmount=2

**DM hybrids:**
- intercept: branch="defend", secondaryBranch="movement", actionType="dodge", hexRange=1, statBonus="reduction", statBonusAmount=2
- phaseout: branch="movement", secondaryBranch="defend", actionType="move", hexRange=2, statBonus="movement", statBonusAmount=2
- shield-advance: branch="defend", secondaryBranch="movement", actionType="dodge", hexRange=0, statBonus="reduction", statBonusAmount=2

### T3 skill metadata
All T3 skills: statBonusAmount=3, tier=3. Branch assignment follows the zone:
- Deep Attack: branch="attack"
- Deep Defend: branch="defend"
- Deep Move: branch="movement"
- AD Boundary: branch based on dominant parent (attack or defend)
- AM Boundary: branch based on dominant parent (attack or movement)
- DM Boundary: branch based on dominant parent (defend or movement)
