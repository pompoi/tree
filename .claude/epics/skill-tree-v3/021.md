# Task: Hex card patterns for new skills

## Description

Create `HexCardPattern` entries in `hex-patterns.ts` for all 33 new skills (9 cross-branch T2 + 24 T3). Remove the 6 old T3 patterns (executioner, whirlwind, mirror-guard, fortress, phantom-step, overrun). Each pattern visualizes the skill's mechanic on a mini hex grid.

## Acceptance Criteria

- [ ] 6 old T3 patterns removed (executioner, whirlwind, mirror-guard, fortress, phantom-step, overrun)
- [ ] 9 cross-branch T2 patterns added (counterstrike, bait-and-punish, step-through, blitz, hit-and-run, ambush, intercept, phaseout, shield-advance)
- [ ] 24 T3 patterns added (all skills from PRD FR-1.3)
- [ ] Every pattern has: skillId, cells array, arrows array, note string
- [ ] Cell types used appropriately: player (white), damage (red), movement (green), block (cyan), conditional (yellow dashed)
- [ ] Arrows use correct styles: facing, move, attack, counter
- [ ] Animation delays set for sequential cell reveals
- [ ] `HEX_PATTERN_MAP` export includes all 57 skills
- [ ] No duplicate skillIds in the patterns array

## Technical Details

### Files affected
- `src/data/hex-patterns.ts` — Add 33 patterns, remove 6 old patterns

### Implementation approach

Design each pattern to visually communicate the skill's mechanic:

**Cross-branch T2 pattern design principles:**
- Show elements from BOTH parent branches
- Use mixed cell types (e.g., counterstrike shows block + damage cells)
- Arrows indicate the flow: block → strike, move → attack, etc.

**T3 pattern design principles:**
- More complex patterns (3-5 cells typical for T3)
- Show the "big moment" of the skill
- Use conditional cells for situational effects
- Animation delays create a storytelling sequence

**Example pattern (Counterstrike):**
```typescript
{
  skillId: "counterstrike",
  cells: [
    { coord: { q: 0, r: 0 }, type: "player" },
    { coord: { q: 1, r: 0 }, type: "block", animationDelay: 1 },
    { coord: { q: 1, r: 0 }, type: "damage", animationDelay: 2 },
  ],
  arrows: [
    { from: { q: 1, r: 0 }, to: { q: 0, r: 0 }, style: "attack" },
    { from: { q: 0, r: 0 }, to: { q: 1, r: 0 }, style: "counter" },
  ],
  note: "Block then counter-strike simultaneously",
}
```

### Organize by section
Group patterns with clear comments:
