# Task: YOMI matchup tables for new skills

## Description

Update the `SKILL_BEATS` and `SKILL_LOSES_TO` maps in `UnifiedSkillGraph.tsx` to include all 33 new skills. Each skill's `interactionNotes` field in the skill data describes its matchup rules. These tables power the opacity-based hover highlights.

## Acceptance Criteria

- [ ] `SKILL_BEATS` map includes entries for all 33 new skills
- [ ] `SKILL_LOSES_TO` map includes entries for all 33 new skills
- [ ] Removed skills (executioner, whirlwind, mirror-guard, fortress, phantom-step, overrun) are removed from both maps
- [ ] Matchup entries are consistent: if A beats B, then B loses to A
- [ ] Base D>A>M>D triangle still works as default fallback
- [ ] Cross-branch hybrids carry matchup rules from both parent branches
- [ ] Hovering a new skill correctly dims/brightens related skills

## Technical Details

### Files affected
- `src/components/skill-tree/UnifiedSkillGraph.tsx` — Update SKILL_BEATS and SKILL_LOSES_TO maps

### Implementation approach

1. Remove entries for 6 deleted T3 skills from both maps
2. Add entries for each new skill based on its `interactionNotes`:

**Cross-branch T2 matchup derivation:**
- Counterstrike (power-hit + parry): beats melee-attack, quick-jab; loses to power-hit, feint, flanking
- Bait and Punish (feint + brace): beats power-hit, lunge; loses to feint, move-based skills
- Step-Through (quick-jab + sidestep): beats melee-attack, power-hit; loses to brace, anticipate
- Blitz (power-hit + dash): beats defend, parry, pivot; loses to brace, sidestep
- Hit and Run (quick-jab + pivot): beats melee-attack, power-hit; loses to brace, flanking
- Ambush (feint + flanking): beats defend, parry; loses to brace, pivot, anticipate
- Intercept (parry + flanking): beats flanking, melee-attack; loses to feint, power-hit
- Phaseout (sidestep + dash): beats all melee ATK; loses to feint, anticipate
- Shield Advance (brace + pivot): beats flanking, melee-attack; loses to feint, charge

**T3 matchups follow their skill descriptions** — each interactionNotes describes beats/loses relationships.

3. Verify bidirectional consistency: write a check that for every `A beats B`, there exists `B loses to A`
