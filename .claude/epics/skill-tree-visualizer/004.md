# Task: Node Interaction + Skill Cards

## Description

Add interactivity to the skill tree: hover to preview a skill card, click to unlock/lock skills (with prerequisite validation). Build the SkillCard component at poker-card proportions, the Tooltip positioning system, and node visual state management (locked/available/unlocked/base). This task bridges the static tree (Task 3) with the persistent state (Task 5).

## Acceptance Criteria

- [ ] `SkillCard` component renders at poker-card proportions (2.5:3.5 / 63mm x 88mm ratio)
- [ ] Card displays: name, tier badge (I/II/III or star for base), branch icon, action type, hex range, description, stat bonus, cooldown, prerequisite names
- [ ] Card header uses branch-colored gradient
- [ ] `Tooltip` component positions the card near the cursor, clamped to viewport edges
- [ ] Hovering a `SkillNode` shows its `SkillCard` in the tooltip
- [ ] Moving mouse away hides the tooltip
- [ ] `SkillNode` visual states:
  - **Base**: always filled, distinct border, non-removable
  - **Unlocked**: branch-colored fill, solid border
  - **Available**: transparent fill, white dashed border (prerequisites met but not yet unlocked)
  - **Locked**: gray fill, gray border, reduced opacity (prerequisites not met)
- [ ] Clicking an **available** node unlocks it (turns to unlocked state)
- [ ] Clicking an **unlocked** node locks it (only if no dependent skills are currently unlocked)
- [ ] Clicking a **locked** node does nothing (or shows a subtle shake animation)
- [ ] Clicking a **base** node does nothing
- [ ] `SkillEdge` states update:
  - Active (both endpoints unlocked): solid, full opacity
  - Available (source unlocked, target not): dashed, medium opacity
  - Inactive (source locked): faded, low opacity
- [ ] `use-skill-tooltip` hook manages tooltip visibility and active skill
- [ ] TypeScript compiles, interactions work in browser

## Technical Details

### SkillCard Layout (poker-card ratio)
```
┌─────────────────────────┐
│ [Icon] Skill Name  [Tier]│  ← CardHeader (branch gradient bg)
├─────────────────────────┤
│ Action: Melee   Range: 1 │  ← CardBody top
│                          │
│ Description text that    │
│ explains what the skill  │
│ does in combat.          │
│                          │
│ +1 Damage    CD: 0      │  ← Stat bonus + cooldown
├─────────────────────────┤
│ Requires: Melee Attack   │  ← CardFooter (prerequisites)
└─────────────────────────┘
```

### Tooltip Positioning
- Portal to document body (avoid SVG clipping)
- Position: offset from cursor by ~16px
- Clamp: if card would extend beyond viewport, flip to other side
- z-index: above all SVG content

### Interaction State (local state for now, wired to Zustand in Task 5)
- For this task, use a local `useState<Set<string>>` to track unlocked skills
- Initialize with `BASE_SKILL_IDS`
- `canUnlock` and `canRemove` from `src/lib/graph-utils.ts` for validation
- Task 5 will replace this local state with the Zustand store
