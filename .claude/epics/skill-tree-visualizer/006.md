# Task: Build Panel Sidebar

## Description

Build the right sidebar UI with 5 save slot buttons, an editable build name, stat summary, ordered decision log, and build management actions (reset, print navigation). Wire the tree's local interaction state (from Task 4) to the Zustand store (Task 5), making the full system persistent. This is the integration task that brings data, tree, and state together.

## Acceptance Criteria

- [ ] `BuildPanel` renders as a fixed-width sidebar (w-80) on the right side of the page
- [ ] `BuildSlotPicker` shows 5 slot buttons, active slot highlighted with branch-dominant color
- [ ] Clicking a slot switches the active build (tree updates to reflect that build's unlocked skills)
- [ ] `BuildNameEditor` allows inline editing of the build name (click to edit, Enter/blur to save)
- [ ] Stat summary displays: `Damage: +N`, `Movement: +N`, `Reduction: +N` derived from unlocked skills
- [ ] `DecisionLog` shows ordered list of skill picks:
  - Each entry: skill name, tier badge, branch-colored dot
  - Most recent pick at the bottom
  - Tower level number (1, 2, 3...) next to each entry
- [ ] "Reset Build" button opens a confirmation `Modal` before clearing non-base skills
- [ ] "Print Cards" button navigates to `/print`
- [ ] Tree interaction (Task 4) now reads/writes from Zustand store instead of local state
- [ ] `Button` and `Modal` UI primitives are reusable
- [ ] Layout: tree takes remaining space (`flex-1`), sidebar is fixed-width
- [ ] TypeScript compiles, full workflow works: pick skills → see in log → switch slots → persist

### Zustand Integration
Replace the `useState<Set<string>>` in `SkillTreeSVG` (from Task 4) with:
```typescript
const { builds, activeSlot, unlockSkill, lockSkill } = useBuildStore();
const activeBuild = builds[activeSlot];
const unlockedSkillIds = new Set(activeBuild.unlockedSkillIds);
```

### BuildSlotPicker
- 5 buttons in a row
- Active: filled with dominant branch color of that build
- Inactive: outline style
- Empty builds show "Empty" or the default name

### DecisionLog
- Maps `activeBuild.decisionLog` to a list
- Each entry looks up skill from `SKILL_MAP` for display data
- Shows tower level as a number badge
