# Task: Static SVG Tree

## Description

Build the circular skill tree SVG visualization. This includes the root SVG container, 4 concentric tier rings, 3 branch slice separators with labels, skill nodes at their computed polar positions, and prerequisite edge paths. At this stage the tree is read-only — no interaction (that's Task 4).

## Acceptance Criteria

- [ ] `SkillTreeSVG` renders a centered 800x800 viewBox SVG
- [ ] `TierRing` renders 4 concentric dashed circles at radii 60, 160, 260, 360
- [ ] `BranchSlice` renders 3 separator lines from center to outer ring at 60deg, 180deg, 300deg (between slices)
- [ ] Branch labels visible: "Attack" (top), "Movement" (bottom-right), "Defend" (bottom-left)
- [ ] `SkillNode` renders a circle for each of the 31 skills at the correct computed position
- [ ] Nodes are colored by branch: Attack=#ef4444, Movement=#06b6d4, Defend=#22c55e
- [ ] Base skills (Tier 0) have distinct styling (e.g. thicker border)
- [ ] Cross-branch skills render on the border between their two branch slices
- [ ] `SkillEdge` renders quadratic bezier curves connecting prerequisite → dependent skill
- [ ] Edges use a lighter shade of the dependent skill's branch color
- [ ] All 31 nodes visible without overlapping
- [ ] Skill names visible as text labels near or inside nodes
- [ ] Tree centered on the page with dark background
- [ ] TypeScript compiles, `bun dev` renders tree

## Technical Details

### SVG Structure
```tsx
<svg viewBox="-400 -400 800 800" className="w-full h-full">
  {/* Background */}
  {/* Tier rings */}
  {tierRings.map(ring => <TierRing key={ring.tier} radius={ring.radius} />)}
  {/* Branch separators */}
  {branchSlices.map(slice => <BranchSlice key={slice.branch} ... />)}
  {/* Edges (rendered before nodes so nodes are on top) */}
  {edges.map(edge => <SkillEdge key={edge.id} ... />)}
  {/* Nodes */}
  {nodes.map(node => <SkillNode key={node.skill.id} ... />)}
</svg>
```

### SkillNode Component
- SVG `<g>` containing `<circle>` + `<text>`
- Circle radius: ~20px for Tier 0, ~16px for others
- Fill: branch color (always filled at this stage — interaction states come in Task 4)
- Text: skill name, truncated if needed, font-size ~8px

### SkillEdge Component
- SVG `<path>` using quadratic bezier: `M x1 y1 Q cx cy x2 y2`
- Control point offset perpendicular to the midpoint for visual curve
- Stroke: lighter shade of dependent skill's branch color, ~1.5px width

### BranchSlice Component
- SVG `<line>` from (0,0) to (sin(borderAngle)*400, -cos(borderAngle)*400)
- Border angles at 60deg, 180deg, 300deg (halfway between branch centers)
- Branch label positioned at branch center angle, outside the outer ring
