# Task: Data Model + Skill Definitions

## Description

Define all TypeScript types, implement the complete 31-skill data set as a single source of truth, build the polar coordinate layout engine, derive the graph from prerequisites, and write graph utility functions (canUnlock, canRemove). This is the data foundation that both the SVG tree (Task 3) and state management (Task 5) depend on.

## Acceptance Criteria

- [ ] `src/types/skill.ts` defines: `Branch`, `Tier`, `ActionType`, `StatBonus`, `Skill` interface
- [ ] `src/types/graph.ts` defines: `SkillNode`, `SkillEdge`, `SkillGraph`
- [ ] `src/types/player.ts` defines: `PlayerBuild`, `DecisionEntry`, `BuildSlot`
- [ ] `src/data/skills.ts` contains all 31 skills with correct prerequisites, branches, tiers, and stats
- [ ] `SKILL_MAP` (Map<string, Skill>) and `BASE_SKILL_IDS` (string[]) exported
- [ ] `src/data/layout.ts` implements `getSkillPosition(skill)` returning `{ angle, radius }`
  - Attack slice centered at 0deg (top), Movement at 120deg, Defend at 240deg
  - Cross-branch skills positioned at averaged angle of their two branches
  - Tier radii: 0=60, 1=160, 2=260, 3=360
- [ ] `src/data/graph.ts` implements `buildSkillGraph()` deriving nodes (with positions) and edges
- [ ] `src/lib/polar.ts` implements polar↔cartesian conversion
- [ ] `src/lib/graph-utils.ts` implements:
  - `canUnlock(skillId, unlockedSet)` — all prerequisites in set
  - `canRemove(skillId, unlockedSet)` — no dependents in set
  - `getPrerequisites(skillId)` and `getDependents(skillId)`
- [ ] Graph has exactly 31 nodes
- [ ] Edge count matches total prerequisites across all skills (~40 edges)
- [ ] All TypeScript compiles without errors
