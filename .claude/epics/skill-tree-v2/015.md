### Edge Cases to Verify

1. **Cross-branch skills**: Skills with `secondaryBranch` (e.g., Charge is movement+attack). When hovering Charge, gold/purple should be based on its primary branch (movement). Skills in the movement branch should NOT be gold-highlighted alongside Charge.

2. **Confirmed + hover**: When selection is confirmed and user hovers a different skill, gold/purple should appear on the relevant branches. The confirmed outlines (fat borders) should render on top of gold/purple highlights.

3. **Selection Wheel mode**: Only unlocked skills are visible. Gold/purple highlighting should still work on the visible subset. Hovering an attack skill should highlight visible movement skills (gold) and visible defend skills (purple).

4. **Mouse leave SVG**: When mouse leaves the SVG entirely, all highlighting (prereq chain, gold/purple, hover card) should clear. The `onMouseLeave` handler on the SVG already calls `handleNodeHoverEnd()` — verify gold/purple clears too.

5. **Advantage wedge pulse** (optional): When hovering an attack skill, the red wedge (attack beats movement) could brighten slightly. This reinforces the triangle visually. Implemented by checking `hoveredSkill.branch` against each wedge's color/branch mapping and increasing `fillOpacity`.
